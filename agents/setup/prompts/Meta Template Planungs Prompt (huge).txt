DEIN AUFTRAG IN DIESEM RUN
Du arbeitest im **Analysis Mode**. Du erzeugst:
1. einen kompakten **RUN PLAN** für diesen Run
2. ein **AGENT BACKLOG**, in dem du Aufgaben für Claude, Gemini, Codex, Copilot und Human so vorbereitest, 
   dass sie im nächsten Schritt direkt loslegen können.
3. optional: eine Liste von **Memory-Kandidaten für NEXUS**, NUR als Vorschlag.

---

SCHRITT 1 – Systemkontext einlesen
- Lies (gedanklich) NEXUS, Knowledge Hub, Shared Working Memory und die relevanten Governance-Policies.
- Nutze sie als harte Constraints; sie stehen über deinem Reasoning.
- Ziehe bei Bedarf Infos zu aktueller Repo-Struktur, Services, Tests, Infra etc. aus dem Repo.

SCHRITT 2 – RUN PLAN erzeugen
- Erzeuge in deiner Antwort eine Sektion:

  ## RUN PLAN – {KURZER TITEL FÜR DIESEN RUN}

- Max. 10 Bullet-Points, u.a.:
  - aktueller technischer Status (sehr kurz)
  - wichtigste offenen Epics / Baustellen für diesen Run
  - klarer Fokus: Was wird in diesem Run realistisch bearbeitet?

SCHRITT 3 – AGENT BACKLOG mit To-Dos für alle
- Danach eine Sektion:

  ## AGENT BACKLOG

- Erstelle eine Markdown-Tabelle mit Spalten:
  Epic | Task | Owner (Claude/Gemini/Codex/Copilot/Human) | Prio (P1–P3) | Status (Open/Planned/Done)

- Für jede relevante Baustelle, die du erkennst ({OPTIONAL: STICHWORTE VOM USER}), definiere mindestens:
  - 1 Task für **Claude**  
    (Planung, Spezifikation, Checks, Dokumentation – alles im Rahmen der Write-Gates)
  - 1 Task für **Gemini**  
    (Audit, Governance-/Architektur-Review, Konsistenz-Checks, Risiko-/Edge-Case-Analyse)
  - 1 Task für **Codex**  
    (konkrete Code-Umsetzung nach deiner Spezifikation; keine Governance-Edits, keine Autonomie)
  - optional Tasks für **Copilot**  
    (Assistenz, Feinschliff, Tests erweitern, Boilerplate, Komfort)
  - mindestens 1 Task mit Owner **Human**  
    (Entscheidungen, Freigaben, manuelle Anpassungen, Git/Infra-Tasks)

- Markiere Aufgaben, die Agenten übergeben werden sollen, explizit als Handoffs im Stil des Knowledge Hubs:
  `[OPEN] Gemini → Codex: ...`, `[OPEN] Claude → Human: ...` etc.

- Wenn du für einen Agenten in diesem Run KEINEN sinnvollen Task siehst, schreibe explizit
  eine Zeile wie:  
  `Meta | Aktuell keine sinnvolle Aufgabe erkannt | Codex | P3 | Open`

SCHRITT 4 – Optionale MEMORY-KANDIDATEN FÜR NEXUS
